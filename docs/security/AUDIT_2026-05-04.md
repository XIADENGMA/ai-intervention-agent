## How to read this report

Each finding is scored along **two axes**:

1. **Vendor severity** — what the upstream advisory says. Untouched.
2. **Effective risk for `ai-intervention-agent`** — does our code actually
   reach the vulnerable path, and does our deployment posture (loopback by
   default, no public file uploads, no OAuth flows) neutralise it?

The **action column** is what a future maintainer should actually do. In
most cases that is "wait for Dependabot", because every advisory has an
upstream fix already shipped and our `dependabot.yml` is configured to open
weekly PRs.

## Why not just upgrade everything now?

We considered manually bumping `fastmcp 3.1.1 → 3.2.4` (which would cascade
the Starlette / python-multipart / cryptography / authlib upgrades for
free). The release notes show two breaking changes:

1. **3.2.0** — `Route app tool calls via ___-prefixed names`. Only affects
   users of `FastMCPApp`. We use plain `FastMCP`, so we are unaffected.
2. **3.2.4** — `Scope tasks to authorization context, not session`. Only
   affects users of background tasks behind auth. We don't.

So the upgrade *is* safe from an API-compat standpoint, but the rationale
for not jumping the gun is:

- All findings above are **already mitigated** by our deployment posture.
  There is no smoking gun.
- Dependabot is configured (`.github/dependabot.yml`) to open weekly
  grouped PRs. Letting that flow do the work is more reviewable than a
  hand-coordinated bump that mixes 5+ packages in one commit.
- The next dependency bump is a natural fit for a v1.5.22 / v1.6.0 release
  cycle, which we will cut whenever we accumulate enough additional
  product-level changes to justify it.

If a maintainer disagrees and wants to pull the security wave forward, the
recommended sequence is:

```bash
uv lock --upgrade-package fastmcp \
        --upgrade-package markdown \
        --upgrade-package pygments \
        --upgrade-package python-dotenv \
        --upgrade-package python-multipart \
        --upgrade-package starlette \
        --upgrade-package cryptography \
        --upgrade-package authlib \
        --upgrade-package werkzeug
uv sync
uv run python scripts/ci_gate.py --with-vscode
uv run pip-audit
```

Then verify the audit count drops from 17 to 0 and run a quick web UI
sanity check by booting `uv run python web_ui.py --port 18099` and curling
`/api/health`.
