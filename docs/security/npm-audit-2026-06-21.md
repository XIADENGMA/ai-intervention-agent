# npm dependency security triage - 2026-06-21

## Status

Partially remediated with a documented dev-tool exception.

- Remediated: the original moderate findings for `brace-expansion` and `js-yaml` are closed in `package-lock.json`.
- Remaining: `npm audit --audit-level=moderate --json` reports 4 findings through `@vscode/test-cli@0.0.12 -> mocha@11.7.5`.
- Runtime exposure: none observed. The remaining packages are in the VS Code extension test runner dependency tree and are not included in the extension package.

## Rejected Remediation

`npm audit fix` recommends `@vscode/test-cli@0.0.11`. This was tested and rejected:

- It did not clear the audit; `diff`, `serialize-javascript`, and Mocha remained reportable.
- It pulled older deprecated dependencies such as `glob@7.2.3` and `inflight@1.0.6`.
- It is a downgrade of the VS Code test runner and increases maintenance risk without improving the audit result.

Nested npm `overrides` for the Mocha subtree were also tested and did not change the installed/locked Mocha child dependencies. The root security overrides remain in place because they still guard packages npm can override successfully.

## Follow-up Trigger

Revisit this exception when any of the following is true:

- `@vscode/test-cli` releases a version newer than `0.0.12`.
- Mocha releases a stable version outside the audited vulnerable aggregate range and `@vscode/test-cli` adopts it.
- npm overrides begin resolving Mocha's `diff` and `serialize-javascript` child dependencies without producing invalid dependency trees.
- The extension package starts bundling test-runner dependencies, which would change the runtime exposure assessment.
