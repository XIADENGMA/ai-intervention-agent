### #C `_classifyFetchError` 5-rule 完整性扩展

R289 covered 5 类，但常见 fetch error 还有：

- HTTP 4xx (403 forbidden / 401 unauthorized) — 当前归到 `serverResponseInvalid`
  但用户应做的事是 "重新登录" 而非 "联系运维"
- HTTP 503 / 502 (服务降级) — 应建议 "5 秒后重试" 而非 "联系运维"
- CORS preflight 失败 — `TypeError` 但用户无能为力

建议第 6/7 类分类：`status.unauthorized` (401/403) +
`status.serviceUnavailable` (503/502)，需要从 response.status 解析。

estimated: 1 commit, 8-10 invariant tests, 0.5 cycle 日。

### #D Glass / Composer 平台特性深挖

R288 的 README sequenceDiagram 是 "viewable + abstract"，但 Glass / Composer
平台还有更深的集成机会：

- Cursor Glass "background agent" mode → AIIA 多 task 队列是否能自动收尾
- Composer "compose mode" → 是否能跳过 `auto_resubmit_timeout` 让 AIIA
  pause 直到用户回到聊天
- VS Code 插件 webview 是否能直接渲染 AIIA UI 而非 iframe (性能 + 主题
  一致)

estimated: 1-2 cycle 日 研究 + 1-3 cycle 日实现。

### #E 错误消息 i18n key 命名约定

sub-namespace：

estimated: 1 commit, 5 key rename × 4 locale = 20 改动 + 2 处豁免 sync, 0.5 cycle 日。
