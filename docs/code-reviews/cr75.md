### 2.1 Pydantic field validator coverage 深化期完全工业化 (R396 + R402)

cycle-43 启动 (R380 1st + R384 2nd 巩固) → cycle-44 (R388 3rd 工业化) →
cycle-45 (R396 4th 深化 + R402 5th 完全工业化), 单维度 0→5 应用跨 3 cycle
完成全 lineage:

| Pass | R#   | 应用对象                                                                    | 识别方式                          |
| ---- | ---- | --------------------------------------------------------------------------- | --------------------------------- |
| 1st  | R380 | NotificationConfig                                                          | 命名 pattern (suffix)             |
| 2nd  | R384 | Task                                                                        | inline comment 显式 clamp         |
| 3rd  | R388 | WebUIConfig + FeedbackConfig                                                | 命名 pattern (bare 形式)          |
| 4th  | R396 | NotificationEvent                                                           | 真实 bug fix + 源同步             |
| 5th  | R402 | NotificationSectionConfig + WebUISectionConfig + FeedbackSectionConfig (×3) | **双风格** (decorator + Annotated) |

**R396 真实价值 — 源同步修复**:
`NotificationEvent` 是 Pydantic 模型, 直接构造或反序列化绕过 manager 兜底
clamp。R396 添加 `@field_validator` 实施 `[0, 100]` clamp:
- 防 `retry_count = -1` → `while retry_count < max_retries` 死循环 (条件永远
  true)
- 防 `max_retries = 99999` → 一个失败通知 spam 9 分钟 + 占满 retry queue 阻
  塞其他 event

R396 不仅是 invariant test, 更是**真实 bug fix** — 这是 cycle-45 唯一一个
"audit → 发现真问题 → 源端修复"的 commit, 与 R324 / R325 lazy-load mock
safety bug fix 同等价值。

**R402 双风格识别突破**:
之前 R380/R384/R388/R396 都只识别 `@field_validator` decorator 风格, R402
扩展同时识别 `Annotated[X, BeforeValidator(func)]` Pydantic v2 替代风格:

```python
def _detect_validator_style(class_def: ast.ClassDef, field_name: str) -> str | None:
    if _has_field_validator_decorator(class_def, field_name):
        return "decorator"  # @field_validator
    if _has_annotated_validator(class_def, field_name):
        return "annotated"  # Annotated[X, BeforeValidator(...)]
    return None
```

`shared_types.NotificationSectionConfig/WebUISectionConfig/FeedbackSectionConfig`
共 11 数值字段全部用 `Annotated` 风格, R402 audit 全 pass。

**特殊价值**: 防 future contributor 用 Annotated 风格写新字段时 R388
audit 漏检 — invariant 自身的双风格识别是元 invariant (R402 invariant 的
invariant 是 "validator 风格枚举的完整性")。

**建议**: cycle-46 或 cycle-47 引入**新维度首次落地**, 候选 (按可行性排序):
1. **Flask blueprint registration matrix invariant** — 锁所有 blueprint 必
   须在 web_ui.py 注册, 每个 blueprint 必须有 url_prefix + 唯一名字 (从
