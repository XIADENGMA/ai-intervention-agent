## 2. R331 — v3.9 async race contract 深化期启动

cycle-35 R326-R330 完成了 v3.9 pattern 的 "**锁定 what**" (lock 顺序 +
白名单 + reentry 禁用)。R331 是 v3.9 进入 **深化期** 的标志, 从 "锁定
what" 升级到 "**锁定 why**":

| Aspect             | cycle-35 (R326-R330)                  | cycle-36 R331 (深化)                |
| ------------------ | ------------------------------------- | ----------------------------------- |
| 锁定层级           | 锁存在 + 类型 + 数量 + 顺序           | + **rationale 注释可执行验证**      |
| reentry 必要性     | 隐式 (RLock 白名单接受任何理由)       | **显式 reentry chain 静态证明**     |
| 防御性 vs 必要性   | 不区分                                | **明确分类** (hard requirement vs 防御性) |
| audit 维度         | 锁站点                                | + **rationale comment 关键字**      |

**R331 关键发现**: 通过 AST 追踪发现 ``_FEEDBACK_TIMEOUT_CALLBACK_LOCK``
存在**真实的 reentry chain**:

- ``_ensure_feedback_timeout_hot_reload_callback_registered`` 持锁状态下
  调用 ``_sync_existing_tasks_timeout_from_config()`` (web_ui.py)
- 后者也 ``with _FEEDBACK_TIMEOUT_CALLBACK_LOCK:`` (web_ui_config_sync.py:47)
- 这是 RLock 的**硬性需求**, Lock 会 self-deadlock

此前从未被显式静态证明 — R331 让此契约**可被 CI 强制**。
