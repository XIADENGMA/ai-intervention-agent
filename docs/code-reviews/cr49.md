### §2.2 新模式 (cycle-4 + cycle-5 沉淀，共 6 个 P)

cycle-4 已沉淀 4 个 P (P1-P4)。cycle-5 沉淀 2 个新 P：

**P5 (cycle-5 Track B): self-test 元 invariants**

> 一个 invariant test 也是代码 → 也可能有 bug → 它应当**测试自己**。

cycle-5 Track B 加了 4 个 self-test 元 invariants 守护主扫描逻辑。
这是 cycle-4 P2"测试 fail ≠ 代码 bug" 的 **direct application** —
不光要怀疑测试，还要主动写 test-of-test 守住测试的正确性。

应用场景：
- 任何用复杂 regex / parsing / AST 的 invariant
- 任何检测"模式 X 在 codebase 出现/不出现" 的 invariant
- 任何会因为 false-positive 错怪代码的 invariant

**P6 (cycle-5 Track C/D): "audit ≠ uniformity"**

> 修一个一致性 invariant 时，**显式保留**有 semantic 理由的例外。

cycle-5 Track C 保留 4 个 ad-hoc focus ring 组件，cycle-5 Track D 保留
R65 `.btn:active` 的 `#d97757`/`#c56a4c`。这两个例子都不是"漏改"，
是**有 trade-off 决策**的显式保留：

- focus ring 色作为 semantic differentiator（pwa-install 紫，ios 紫，
  countdown 暖/冷）是合法设计选择
- R65 `:active` 保留品牌橙让品牌色还能"瞬间出现"，否则 light theme
  完全失去 Anthropic Orange 痕迹

把"刻意保留的例外"写进 commit message + 测试注释，避免后续 audit
误以为是漏改。
