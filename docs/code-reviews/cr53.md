is the canonical `uv run` substitute. `shutil.which()` is std-library
and zero-runtime-cost.

### §2.3 `981998c` — R271 OpenAPI `maxLength` drift fix (exploratory Track E)

**File**: `src/ai_intervention_agent/web_ui_routes/notification.py` (+
`tests/test_feat_correctness_cycle23_prompt_maxlength_consistency_r271.py`
new).

**Change**: `feedback_config` POST endpoint OpenAPI schema:
```diff
- resubmit_prompt:
-   type: string
-   maxLength: 10000
+ resubmit_prompt:
+   type: string
+   maxLength: 100000
+   description: 自动重发提示语；最大长度与 server_config.PROMPT_MAX_LENGTH 对齐（100,000 字符）

- prompt_suffix:
-   type: string
-   maxLength: 10000
+ prompt_suffix:
+   type: string
+   maxLength: 100000
+   description: 反馈后缀提示语；最大长度与 server_config.PROMPT_MAX_LENGTH 对齐（100,000 字符）
```

**Rationale**: 4-way schema drift:
- `server_config.PROMPT_MAX_LENGTH` = 100,000
- `web_ui.html` `maxlength="100000"`
- `webview.ts` (VSCode) `maxlength="100000"`
- `notification.py` OpenAPI schema `maxLength: 10000` ← **OUTLIER**

Real user impact: SDK / Swagger-codegen clients consuming the OpenAPI
spec would silently truncate user input at 10K, breaking the 90K of
extra capacity the backend + UI both promise. Discovery scenario: user
opens Swagger UI / generates API client → sees 10K limit → reports
"prompts truncating prematurely" bug.

**Risk**: **low-medium**. Schema-only change; runtime semantics
unchanged (backend was already accepting 100K). One unknown: if a
client app explicitly relied on the 10K limit as a validation cue,
they now see 100K — but this is an additive widening, not a breaking
narrowing.

**Test surface**: 5 new invariants in
`test_feat_correctness_cycle23_prompt_maxlength_consistency_r271.py`:
- `server_config.PROMPT_MAX_LENGTH == 100000` (semantic anchor)
- `web_ui.html` template has `maxlength="100000"` for both fields
- `webview.ts` has `maxlength="100000"` for both fields
- `notification.py` OpenAPI schema declares `maxLength: 100000` for both
- All 4 sources cross-platform consistent (single source of truth check)

### §2.4 `00ee9cf` — R272 ESC handler delegation (exploratory Track F)

**File**: `src/ai_intervention_agent/static/js/keyboard-shortcuts.js` (+
3 minified/compressed siblings + `test_feat_a11y_cycle23_escape_handler
_delegation_r272.py` new).

**Change**: ESC keyboard shortcut handler:
```diff
- if (settingsPanel && settingsPanel.classList.contains('show')) {
-   settingsPanel.classList.remove('show');
-   settingsPanel.classList.add('hidden');
-   return;
- }
- if (imageModal && imageModal.classList.contains('show')) {
-   imageModal.classList.remove('show');
-   return;
- }
+ if (settingsPanel && settingsPanel.classList.contains('show')) {
+   if (typeof settingsManager !== 'undefined' && settingsManager && typeof settingsManager.hideSettings === 'function') {
+     settingsManager.hideSettings();
+   } else {
+     // Fallback: settings-manager.js 未加载
+     settingsPanel.classList.remove('show');
+     settingsPanel.classList.add('hidden');
+   }
+   return;
+ }
+ if (imageModal && imageModal.classList.contains('show')) {
+   if (typeof closeImageModal === 'function') {
+     closeImageModal();
+   } else {
+     // Fallback: image-upload.js 未加载
+     imageModal.classList.remove('show');
+   }
+   return;
+ }
```

**Rationale**: bare `classList.remove('show')` bypasses critical cleanup
that `hideSettings()` and `closeImageModal()` perform:
- **focus restore** (`_previouslyFocusedElement.focus()` — R263a + R264)
- **inert background** (`document.body.inert = false`)
- **overflow restore** (`document.body.style.overflow = ''`)
- **event listener cleanup** (`removeEventListener`)
- **escape modal state machine** (parent close handlers, animation
  triggers)

Without delegation, ESC-closing a settings panel left focus floating in
limbo (no return target), `aria-hidden`/`inert` stuck on background
content (assistive tech reads stale state), and listener leaks across
N open-close cycles.

Discovery scenario: a11y audit + R263a/R264 retrospective — "we capture
focus on open, but do we restore it on **all** close paths?" → grep
all close call sites → ESC handler was the one bypass.

**Risk**: **low**. `typeof` guards preserve backward compat (if those
modules ever fail to load, fallback to old `classList` swap). Pattern
matches the existing `keyboard-shortcuts.js` global-vs-module
defensive style. Test confirms both delegation + fallback annotations.

**Test surface**: 6 new invariants in
`test_feat_a11y_cycle23_escape_handler_delegation_r272.py`:
- ESC handler delegates to `settingsManager.hideSettings()` (string
  match)
- ESC handler delegates to `closeImageModal()` (string match)
- Uses `typeof settingsManager !== 'undefined'` guard
- Uses `typeof closeImageModal === 'function'` guard
- Has `// Fallback:` comment annotations (documentation lock-in)
- Early `return;` after each branch (prevent double-firing)

- `zh-CN.json`: `barkUrl="Bark URL"` (品牌词), `barkDeviceKey="设备 Key"`
- `zh-TW.json`: `barkUrl="Bark URL"`, `barkDeviceKey="裝置 Key"`

**Rationale**: audit of `web_ui.html` for hardcoded labels found these

feature ship (Bark URL + Device Key are the **first** two fields in
the section, so they got typed before the i18n convention was
established for this block).

Real user impact: zh-CN / zh-TW users see English labels in the
otherwise-Chinese Bark settings — visual jarring + reduces trust in
i18n completeness.

**Risk**: **zero**. Pure i18n additive change; English fallback
preserved if locale lookup fails.

**Test surface**: 9 new invariants in
`test_feat_i18n_cycle23_bark_label_hardcoded_r273.py`:
- 4 locales × 2 keys exist (4 tests)
- `web_ui.html` has both `data-i18n` attributes (2 tests)
- Translation values non-empty (2 tests)
- **Meta-lint**: all `<span class="setting-title">` in `web_ui.html`
  must have `data-i18n` attribute (1 test, future-proofing — prevents
  next dev from re-introducing the same oversight)

## §3 Cross-cycle observations

**L1 (negative-finding ROI)**: cycle-23's 3 of 4 follow-up tracks
closed with "**negative confirmation**" (no new bug found, audit
confidence locked in). **This is correct cycle outcome** — finding
zero bugs in a sweep is signal that the original fix (R268, R266) was
narrow + correctly scoped, not just a tip of the iceberg. cr52's L3
"avoid artificially splitting" still holds, but cr53 adds nuance:
**audits with negative findings should still be documented**
(in cr53 §1) so future cycles don't re-audit the same surface.

**L2 (exploratory ROI > planned ROI)**: cycle-23 planned 4 follow-up
tracks (3 negative, 1 yielding R270), but exploratory audit yielded
**3 real bugs** (R271 maxLength drift, R272 ESC delegation, R273 i18n
hardcode). This **reverses cr52's L2 narrative** — planned tracks have
low yield because they re-audit recently-touched code; exploratory
sweeps over **adjacent untouched surfaces** (OpenAPI schema, keyboard
handler, i18n attrs) find latent issues.

**Heuristic update**: in autonomous loop mode, dedicate ~30% of cycle
budget to **untouched-surface exploratory audit** rather than 100% to
following up on the previous cycle's spillover.

**L3 (i18n drift is silent)**: R273 is a particularly insidious bug
class because:
1. Chinese user sees English label → assumes "intentionally untranslated"
   (Bark, Apple, GitHub etc. brand names often left as-is)
2. Developer sees existing pattern (no `data-i18n`) → tendency to
   follow same pattern for new labels
3. No runtime error, no console warning, no test failure (until R273's
   meta-lint)

**Meta-lint pattern** (R273 test 9): not just "lock in this specific
fix" but "lock in a **rule** that auto-detects all future occurrences
of this oversight class". This is methodology v3.4 candidate —
**every i18n / a11y / correctness fix should ship with at least one
meta-lint invariant** that prevents the entire class of bug.

**L4 (cross-platform schema drift)**: R271's 4-way drift (`server_
config.py` ↔ `web_ui.html` ↔ `webview.ts` ↔ `notification.py` schema)
is symptomatic of **distributed-source-of-truth anti-pattern**.
mining-14 candidate: scan project for other multi-source declarations
(e.g., port numbers, default timeouts, retry counts) and centralize
or invariant-test for consistency.

**L5 (delegation > duplication)**: R272 ESC handler shows that
**every modal close path must delegate to a single canonical close
function**. mining-15 candidate: project-wide audit of "modal close
trigger sites" — verify all go through `closeXxx()` / `hideXxx()`
helpers, never via direct `classList` swap.
