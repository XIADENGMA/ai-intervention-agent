### 2.4 真实 spillover (1 个) — R72A test isolation bug

cycle-52 *bonus* 真实 spillover (与 cycle-49 race fix 同类型 — pre-existing
bug 在 xdist 并行下暴露):

#### 现象
- `tests/test_root_logger_intercept_r72a.py::test_install_function_called_at_module_load` 概率性失败 (xdist -n 4)
- 单文件运行不复现, 必须配合 xdist 才暴露

#### 根因
- 测试文件顶部 `from enhanced_logging import InterceptHandler` 拿到旧类引用
- xdist 同一 worker 上前面 test 多次 `importlib.reload(enhanced_logging)`
- 每次 reload 创建 *新* InterceptHandler 类 (Python class identity 按定义时刻区分)
- 当前 test 再次 reload + 安装 *新* 类的 InterceptHandler
- 测试用 *旧* 类 isinstance 检查 → 漏判 → 0 计数 → fail

#### 修复
- 改用 `enhanced_logging.InterceptHandler` (reload 后的最新引用) 做 isinstance
- 详细注释解释 reload isolation 教训

#### 价值
- 与 cycle-49 i18n scanner race-safe fix 同类型 — *测试 isolation bug 在
  xdist 并行下暴露* — 都是 *真实 production 测试基础设施 bug*, 不是 false
  positive
- R72-A invariant 自身设计漏洞 (reload + 跨类 isinstance), 不是 R72-A
  production code 的问题
- 类似 cycle-49 修复, 这也是 invariant *自身需要 meta-invariant 保护* 的
  又一证据 — 未来 cycle 可考虑给 R72A 加 meta-invariant
