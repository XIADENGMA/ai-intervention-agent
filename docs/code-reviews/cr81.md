### 2.2 v3.10.3 OpenAPI error response schema 突破 7 成决胜 threshold

ratchet 节奏 (cycle-51 #A1 推到决胜 threshold):

| Stage   | R#   | Cycle | Coverage         | Ratchet baseline | Δ      |
| ------- | ---- | ----- | ---------------- | ---------------- | ------ |
| 启动    | R422 | 48    | 3/51 = 5.88%     | 0.05             | —      |
| 1st     | R428 | 49    | 11/51 = 21.57%   | 0.15             | +15.69 |
| 2nd     | R432 | 49    | 19/51 = 37.25%   | 0.30             | +15.68 |
| 3rd     | R436 | 50    | 27/51 = 52.94%   | 0.50             | +15.69 |
| **4th** | R440 | **51** | **36/51 = 70.59%** | **0.70**     | **+17.65** |

**7 成决胜 threshold (70.59%)** = 客户端能 *默认假设* error 响应有 schema
(因为 7 成都有), 剩余 30% 多是 special cases 或还在 audit. 这是 OpenAPI
API contract 进入 *production-grade* 的里程碑.

cycle-51 #A1 (R440) 推进焦点: **task.py countdown ops** + feedback.py 429:

| 端点                          | 状态码     | 添加场景                                  |
| ----------------------------- | ---------- | ----------------------------------------- |
| POST /api/tasks/<id>/extend   | 400/404/422/500 | 含 error_code 字段 (`invalid_seconds` 等) |
| POST /api/tasks/<id>/freeze   | 400/404/409/500 | idempotent No-Op 标记 (`already_frozen`)  |
| POST /api/submit              | 429        | 含 retry_after hint (客户端 backoff)      |

task.py countdown ops 是 user-facing 关键交互, error_code 字段帮助客户
端实现 enum-based 错误分类逻辑。feedback.py 429 的 retry_after 让客户
端能正确 backoff 重试.
