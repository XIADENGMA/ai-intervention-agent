## 4. 风险 & 隐患

### 4A. CI 加速带来 test pollution 暴露

R305 引入 pytest-xdist 后, 一些跨文件共享 global state 的测试 (R72-A
root logger handler, R145 NotificationManager class-level state) 在
**特定 worker 调度顺序下** 会暴露 pollution。当前 R305 选 `--dist=loadfile`
缓解此问题 (同文件保持同 worker), 但 cross-file pollution 仍然存在。

**风险等级**: 中
**缓解建议**:
- 短期 (cycle-31): 监控 CI 失败模式, 如果某测试 xdist 失败率 > 5%, 加进
  "需要 serial" 黑名单
- 中期 (cycle-32+): conftest.py 加 autouse fixture 重置 root logger /
  NotificationManager class-level state, 让 cross-file 隔离硬保障
- 长期: 考虑用 `--dist=loadgroup` + `@pytest.mark.xdist_group(name)`
  对受影响测试明确分组

### 4B. v3.6 pattern 工业化后, 新 pattern 提出节奏放缓

cycle-30 全部聚焦 v3.6 pattern 第二应用, 没有提出新 pattern。这在
methodology 演进中是正常的 "巩固期", 但如果连续 3 cycle 不引入新 pattern
可能错失新发现。

**风险等级**: 低
**缓解建议**:
- cycle-31 starter 时检查: 当前 issue 列表 / 用户反馈是否暗示新 pattern
  需求 (e.g. database schema migration / async background task lifecycle)
- 若有新 pattern 候选, cycle-31 不应再做第三轮 v3.6 应用, 而应该提新
  pattern 拿到 v3.7 早期 PoC

### 4C. CI 加速后 fail-fast 表现差

`-n 4 --dist=loadfile` 模式下, 单个测试失败不会立即中断其余 3 worker,
需等所有 worker 完成才输出失败列表。CI fail-fast 体验比串行差 (串行
`-x` 立即停, xdist 不支持)。

**风险等级**: 低 (50s 总时长已经够快, fail-fast 收益有限)
**缓解建议**: 不动, 50s 总时长可接受 (3.76x 加速 > fail-fast 收益)。
