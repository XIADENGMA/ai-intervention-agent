### 3D. R324 (c017de0) — lazy-load mock pollution bug fix (核心成就!)

**bug 类型**: cross-test mock pollution due to lazy-load short-circuit

**bug 现象**:

- `tests/test_web_ui_routes.py::TestBarkTestEndpoint::test_bark_send_fail_
  with_detail_no_status_code` 等多个测试用 `@patch("...NotificationEvent")`
  单一 attribute mock
- 单独跑该测试 → fail; 在某些前置测试触发 lazy load 后 → pass; 完全运
  行顺序敏感的 flake
- 2026/3/27 提交 (4a5629a `:white_check_mark: test: comprehensive unit
  test coverage improvement → 98.73%`), 在 `main` 静默 broken ~2 个月

**根因**:

- `_ensure_notification_loaded()` short-circuit 只看 `if notification_manager
  is None`
- 单 attribute mock (e.g. `@patch("...NotificationEvent")`) 不动
  `notification_manager`
- 如果 `notification_manager` 还是 None (没被前置触发) → short-circuit
  fail → 进入 import 分支 → **覆盖**已 mock 的 `NotificationEvent` 为真实
  class
- 测试断言 `test_event.metadata == {"bark_error": ...}` 失败, 因为
  `test_event` 是真 `NotificationEvent` instance (metadata 只有
  `test_metadata`, 没有 `bark_error`)

**修复**:

- 把 short-circuit 改为 "全部 4 个 attribute 已 non-None 才 early return"
- 把 import 后的赋值改为 per-attribute null check (`if X is None: X = _x`)
- 已 mock 的 attribute 不会被覆盖
- 性能不变: 所有 attribute 共享同一次 import (R20.10 ~43ms cold-start
  saving 保留)

**R324 invariant (13 tests)** 防止 future regression:

- Layer 1 (4): source 函数体含 4 个 per-attribute null check + docstring
  R324 marker + 解释 per-attribute 策略
- Layer 2 (4): mock 每个 attribute 后 mock 不被覆盖 + 全已加载时 early
  return
- Layer 3 (1): bug regression guard 直接复现原始触发场景
- R324 lineage marker (4): R316/R319/R323 prior app 引用 + 4th app marker
  + source-layer extension 关键词

**亮点**:

- v3.8 test-isolation pattern **从 test 层推到 source 层** (4th app), 是
  pattern 跨域应用的关键证明
- 同时修复真实 bug, 不只是 invariant 加固
- 单 commit 价值密度: bug fix + pattern extension + invariant 三重收益
