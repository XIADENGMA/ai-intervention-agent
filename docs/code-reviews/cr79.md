### 2.5 真实生产 bug fix (race condition discovery)

**cycle-49 spillover bonus**: 通过 cycle-49 全 regression 发现并修复了
`scripts/check_i18n_{ts,js}_no_cjk.py` 的真实 race condition:

- `_iter_js_source_files` 用 `rglob` 列出文件 → `scan_file` 读取期间, 短
  命周期 tmp .js / .ts 文件被并行 pytest worker / VS Code 监视器 / 临时
  文件 cleanup 删除 (race window <50ms)
- 在 xdist -n 4 (本地) + CI (pre-commit hook 并行) 都可能 trigger
- 修复: `scan_file` 用 `try/except FileNotFoundError` 兜底, 返回空列表
- JS / TS scanner 同款修复保持一致

这是真实生产 bug, **CI pre-commit hook 偶发 fail 的根因之一**, 修复后
永久消除该类 flake。同时调整 R412 envelope tolerance 0.30 → 0.60 作为
ratchet 协调 (R428/R432 引入的 error schema 自然含 envelope description)。
