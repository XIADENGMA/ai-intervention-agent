**关键观察 #1: sound_volume bidirectional 转换**

`sound_volume` 是 cycle-28 第一个**双向语义不一致**的 const:
- Python 层: `int` 0-100 (`shared_types.py: sound_volume: int = 80`)
- JS 层: `float` 0.0-1.0 (`HTMLMediaElement.volume`)

R292 invariant 锁了 7 个 source site (Python 5 处 80 + JS 2 处 0.8) 并新增
专门的 `test_sound_volume_int_float_conversion_invariant` 测试，确认转换
精度 (80/100=0.8 严格等于) 没有 drift。

**关键观察 #2: bark_url_template 是 cycle-28 第一个 templated string const**

`bark_url_template = "{base_url}/?task_id={task_id}"` 包含两个占位符。R293
特别加了 `test_bark_url_template_placeholder_invariant`，确保模板里**两个
placeholder 都在 (`{base_url}` + `{task_id}`)**，不能漏掉任何一个。这是
cycle-28 引入的新 invariant pattern: **template-content invariant** vs 之前的
value-equality invariant.

**结论**: cycle-28 第三轮 audit 覆盖了 const 审计的两个 **新维度** (bidirectional
+ templated)。methodology v3.6 起 const-audit 应**优先 scan templated string
+ unit-conversion 边界**，因为这两类最容易隐藏 drift。

## 4. 风险 & 隐患

### 4A. 文档维护负担 (R295 引入)

R295 引入了 "Worked examples 表格" 跟踪所有 3 个 dynamic-key case。**未来
每加一个 helper-returned i18n key 系列 (R296+R29x 都可能算)，都要手动更新
这个表**。R295 invariant 锁了"必须列 3 个 case"，没锁"未来新 case 必须
入表"。

**风险等级**: 中
**缓解建议**: cycle-29/30 加一个 meta-invariant 测试 — 扫描 `_WEB_RESERVED_DYNAMIC`
和 docs Worked examples 表格的 **case 数应一致**。但这个 meta-invariant
本身复杂度高 (要识别 "case" 边界)，可能不值得做。**优先方案**: 写一行 review
checklist "添加 dynamic key 时记得加 case 行" 即可。

### 4B. const 审计的边界递减效应

R292/R293 之后，**项目里"明显有 drift 风险的 const"基本审计完毕**。继续
做第四/五轮 audit 边界效用递减明显。下一步应转向：
- **运行时 invariant** (e.g. 启动后 5s 内必须能完成 health check)
- **跨模块协议 invariant** (e.g. notification → web_ui event payload schema 不能 drift)

### 4C. dynamic i18n helper 的扩展边界

`_classifyFetchError` + `_classifyHttpResponse` 一共 7 keys，覆盖 ~80%
错误场景。剩 20% 包括:
- Rate limit (429) — AIIA 本地服务无 rate limit，价值低
- Session expired (419) — AIIA 无 session，价值低
- DNS resolution failure — `fetch()` 上一步就抛 TypeError，已被 `_classifyFetchError` `networkOffline` cover
- CORS — 只在 dev 环境出现，prod 不会触发

**结论**: 错误分类 helper 系列已**接近完备**。未来若新增 dynamic key，应
跨模块（如 `_classifyConfigError(error)`）而非继续扩展现有 helper。
