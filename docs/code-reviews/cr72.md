### 2.5 Security header strict mode — 新维度首次锁定 (R376)

R306 (cycle-31) 锁定 CSP nonce 三层一致性 (response header ↔ template
attribute ↔ JS reader)。R376 扩到 CSP 之外的其他 6 个安全 header:

- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `X-XSS-Protection: 0` (OWASP 推荐显式关闭老 auditor)
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Cross-Origin-Opener-Policy: same-origin`
- `Permissions-Policy` (geolocation/camera/microphone 等显式禁用)

3 层 invariant 同时锁 **source 字面量** + **Flask test_client 实际响
应**, 双重护栏防止开发者:

- 删除 header (看起来无害的重构);
- 弱化值 (e.g., `X-Frame-Options: SAMEORIGIN` 想嵌 iframe → clickjacking
  攻击面);
- 篡改值 (e.g., `Referrer-Policy: unsafe-url` 想方便调试 → URL token 泄漏)。

**Security header strict mode** 成为方法学的第 11 个 pattern 维度。
