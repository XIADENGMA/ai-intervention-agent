## §2 Per-commit analysis

### §2.1 `991a825` — R284 web_timeout 5-source invariant (Track A / t26-1)

**Scope**: 0 行源码改动，纯 invariant 6 tests，引入新 const lock pattern。

**5 sites lock**:

- `shared_types.NotificationSectionConfig.web_timeout` (Pydantic authoritative)
- `config.toml.default` `web_timeout = 5000` (TOML 模板)
- `notification_manager.NotificationConfig.web_timeout = 5000` (cls default)
- `notification_manager.py` × 2 (`__init__` + runtime reload `cfg.get("web_timeout", 5000)` fallback)
- `web_ui_routes/notification.py::normalize_web_timeout` 内
  `notification_config.get("web_timeout", 5000)` fallback

**新 pattern**: R278/R283 是"前端 hardcode 必须等于 backend authoritative"
(防 UI drift)；R284 引入"backend fallback parsers 必须等于 authoritative
Pydantic default" (防 fallback behavior drift)。

**Drift risk**: 改 Pydantic 7000 但忘 config.toml.default → 新用户拿
7000 但 init 后 disk 拷贝的 config 还是 5000；4 个 fallback parser 不
同步 → config 缺 key 时返回旧值。

**R278 → R283 → R284 evolution**:

| Round | Const lock | Source language | New pattern |
|-------|-----------|-----------------|-------------|
| R276 (cycle-24) | 1 | Python + JS | 跨前后端 |
| R278 (cycle-25 t25-2) | 3 | + HTML | UI form attr |
| R283 (cycle-25) | 4 | (same 3 lang) | 5-way mirror |
| R284 (cycle-26 t26-1) | 5 | Python-only | fallback parser |

从 1 const / 2 lang 走到 5 const / 3 lang，并引入 backend-only fallback
pattern。Invariant suite 从 cross-stack 扩展到 intra-stack 多层一致性。

### §2.2 `410b202` — R285 loadConfig + copyCodeToClipboard null guards (Track B / t26-2)

**Approach**: R268/R279/R280 累计 3 个 stale DOM ref 修复 (finally
× 2 + entry/success path × 1)。cycle-26 R285 第四轮，覆盖：

- `loadConfig()` 3 处 success path DOM 访问 (`#description` /
  `#options-container` / `#separator`)
- `copyCodeToClipboard()` entry + 4 处 await-result + setTimeout DOM 访问

**R285 引入新 guard pattern: `isConnected`**:

3 种 guard pattern 各有适用场景：

| Pattern | When to use | Cycle introduced |
|---------|-------------|------------------|
| re-query (`getElementById`) | finally 块需要操作 DOM 且不确定 element 引用是否新鲜 | R268 (cycle-22), R279 (cycle-25) |
| direct null check | entry 阶段 `getElementById` 局部变量，失败 early return | R280 (cycle-25) |
| **`isConnected` check** | closure 引用 caller 传入节点，检查 still attached | **R285 (cycle-26 新)** |

`isConnected` 性能比 re-query 更优 (无 DOM 树查询)，适合 event handler
closure 持有的 button/preElement 引用。

**修复价值**:

- `loadConfig()`: multi-task 切换中 DOM 重渲染时 TypeError 翻成 "Config
  load failed" 误报 — 配置实际成功，UI 失败。R285 修复后用户看到正确
  状态 (silently skip render + console.warn)
- `copyCodeToClipboard()`: 用户点击 copy 后 message bubble unmount 时
  整个 UI restore 链断裂 (success setTimeout 与 catch setTimeout 都
  抛)，console error 无法被用户感知。R285 4 处 isConnected 兜底

**R285 invariant (9 tests)**:

- 3 R285 loadConfig (description / options-container / anchor)
- 3 R285 copyCodeToClipboard (entry null + 4 isConnected + anchor)
- 3 R268/R279/R280 preserve sanity

**Pattern reuse**: `_strip_js_comments()` 从 R280 复用，让 invariant
regex 不误匹配 R285 doc-comment 里的 bad-pattern 示例。

```
# Summary: 13 TODO items, 12 done, 1 open, 1 open without addressing commit
```

**R286 invariant (7 tests)**:

- 脚本存在 + chmod +x (executable bit 检查)
- module docstring 含 R286 anchor + 关键术语
- `--help` / `--md` / `--strict` CLI flags 各自工作
- 主路径 stdout 含 `# Summary:` 行
- **Read-only invariant** (反向检查): 脚本不能含

  写操作 API
