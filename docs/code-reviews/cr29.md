## 1. Cycle theme вЂ” "audit cascades, not just call sites"

The defining lesson of Cycle 16 is **how easily Pattern B (static-grep)
invariants miss cascading semantic bugs**. Two production-impacting
bugs slipped through 4 cycles each:

| Bug                             | Wrong because               | Caught when                      | Time-in-prod |
| ------------------------------- | --------------------------- | -------------------------------- | ------------ |
| Stale `.min.js` served to dev   | Existence в‰  freshness       | R241 commit audit (incidental)   | R234-R241    |
| Modal-inside-container inert    | Pattern B can't trace cascade | R242 audit follow-up (incidental) | R240-R243    |

Both bugs were "user can clearly see something is wrong" (browser
shows old code; modal pops up but can't be interacted with). Both
were missed by automated tests because:

- Pattern B tests verify **what the source says**, not **what the
  browser does**.
- Manual smoke testing stopped at "I see the modal" / "the JS
  file loads" without exercising the user-relevant interaction.

**Cycle 16 doctrine** (codified in R245's docstring):

> "Always trace cascade, not just call site. If the test verifies
> what a function *says*, ask whether the runtime cascade respects
> that contract end-to-end."

This applies to:

- **CSS cascade** (e.g. R234's `:disabled` rule passed but inline
  style override silently won вЂ” fixed by removing inline style).
- **DOM `inert` cascade** (R240/R244 today).
- **Static asset freshness chains** (R226/R242 producer в†”
  static-compression/R243 consumer).
- **i18n key cascade** (a new key in en.json must propagate to
  zh-CN.json AND pre-compressed mirrors AND `_TEMPLATE_CONTEXT`).

The R245 invariant is the prototype of this doctrine: it parses HTML
structure + JS source to model the cascade analytically. Future
Cycle 17+ candidates should imitate the pattern.

### R242 вЂ” silent-stale-`.min` build-time guard (`9e1e90c`)

**What**: Add `check-static-minified-fresh` pre-commit hook running
`scripts/minify_assets.py --check`. Same shape as R226 (precompress
freshness) but on the `.min.{js,css}` chain.

**Why discovered**: While auditing R241's app.js changes, ran
`uv run python scripts/minify_assets.py --check` and got "йњЂи¦ЃеЋ‹зј©:
settings-manager.js / app.js / main.css" в†’ realised that since
**R234** (4 cycles ago), changes to source `.js`/`.css` were NOT
being mirrored to `.min.{js,css}`, but `_get_minified_file()`
**preferred** the stale `.min` files. So R234/R238/R240/R241 had
all shipped stale to local dev.

**Why production-safe but dev-broken**:
- `.gitignore` L267-274 excludes `*.min.{js,css}`; production
  fresh checkouts have NO `.min.*` в†’ `_get_minified_file()` falls
  back to source. **No production impact**.
- Local dev clones that ran `minify_assets.py` once and kept
  developing thereafter: stale forever. **Dev velocity tax**.
- VSCode-extension developers (likely majority): also dev-stale.

**Audit notes**:
- Hook is content-aware via `content_drifts()`. Correctly noted
  in code comments that mtime is unreliable on fresh clones / CI.
- Side-effect of fix: regenerated 3 stale `.min` mirrors in this
  commit. Catch-up cost = 0 (mirrors weren't tracked anyway after
  R226 picked up the source `.br/.gz` mirrors).
- Mirrors R226 pattern. Documented this in the hook's comment.
- **Tradeoff considered & rejected**: removing minification
  entirely. `.min` saves ~5-10% gzipped + ~10-20% parse time;
  removing would be a regression. Kept with R242+R243 guards.

**Health**: Solid. Closes a real bug. Test coverage is Pattern B
(hook presence + companion script + flag) вЂ” appropriate for a
config-shaped invariant.

### R243 вЂ” silent-stale-`.min` runtime guard (`c41fef7`)

**What**: Make `_get_minified_file()` mtime-check the candidate
`.min` against the source. If `mtime(.min) < mtime(source)`, fall
back to source + `logger.warning` once per file pair.

**Why**: R242 closes the producer side, but two paths still
slip stale `.min` past:
1. `git commit --no-verify` bypass.
2. Pre-existing stale `.min` files whose source isn't touched in
   the current commit (R242 `files:` filter is path-scoped).

R243 is **belt-and-suspenders** on the consumer side, mirroring
the symmetry of R226 в†” static_compression_middleware on the
`.gz/.br` chain.

**Audit notes**:
- Cost: one extra `stat()` per static-asset request. Negligible
  vs `send_from_directory`'s own stat overhead. Static assets
  typically Cache-Control: long-lived so most requests never hit
  this anyway.
- Behavioral preservation: explicit `.min.*` request still passes
  through (caller specified); missing `.min.*` falls back as
  before; `OSError` from `stat()` falls back (no 500). All three
  edge cases covered by tests.
- **Pattern A (true runtime test) by design**: tempdir +
  os.utime + `_get_minified_file()` call. Catches semantic
  regressions, not just text changes.
- Dedup state (`_stale_minified_warned`) is class-level set.
  Justified inline comment about per-process dedup. Per-file
  granularity verified by `test_different_files_warn_independently`.
- **Type-system value demonstration**: ty caught
  `Path.stat = _broken_stat` as `invalid-assignment` before
  commit landed. The fix (`cast(Any, Path).stat = вЂ¦`) is
  idiomatic for monkey-patching in tests. R236 ty pre-commit
  hook paid for itself on the second cycle after introduction.

**Health**: Excellent. Pattern A test, defense-in-depth design,
type-system caught a regression.

### R244 вЂ” modal-cascade-self-inert fix (`7359c1d`)

**What**: Replace `container.inert = true` with new helper
`_setContainerSiblingsInert(openModalEl, value)` that iterates
`container.children` and inert-s every child *except* the open
modal. Both `app.js` (top-level fn) and `settings-manager.js`
(class method).

**Why**: R240 was buggy from day one. Both modals
(`#code-paste-panel` at HTML L713, `#settings-panel` at L976)
live **inside** `.container` (verified by HTML parse:
`Modals INSIDE .container: ['code-paste-panel', 'settings-panel']`).
R240's `container.inert = true` propagated through HTML5 cascade
to the modals в†’ silent UX-killer.

**Why discovered**: Auditing aria-hidden / inert overlap for
F-cycle15-8 prompted "if .container.inert and SVG aria-hidden
both true, is one redundant?". The answer required tracing
cascade: realised the modals would also inherit inert.

**Audit notes**:
- Bug was undetected for **4 cycles** (R240 in Cycle 15 в†’ R243
  in Cycle 16, this commit). All Pattern B tests on those
  commits green-stamped the bug. This is the **case in chief**
  for the cycle theme.
- Fix architecture choice: iterating `.children` keeps the
  helper simple and avoids "modal-as-body-sibling" HTML restructure
  (which would have touched many CSS selectors + JS queries).
  Considered & rejected.
- Backwards compatibility: updated R240 + R241 invariant tests
  to accept the new helper as valid (via additional regex in
  `_OPEN/_CLOSE_INERT_PATTERNS`). R240's regression-guard
  semantics preserved.
- **Regression guard added in R244 test**:
  `TestNoRegressionToContainerDirectInert` explicitly forbids the
  old buggy pattern. If anyone re-introduces
  `container.inert = вЂ¦` in a modal path, this test fails. This
  is the "tombstone test" pattern (R211 used the same shape).

**Health**: Fix is correct and minimal. The 4-cycle delay is
shameful; mitigated by R245 generalisation.

### R245 вЂ” HTML cascade-aware structural invariant (`84ae155`)

**What**: New invariant test that parses `web_ui.html` to enumerate
every `role="dialog"` element and its ancestor chain, parses
`app.js` + `settings-manager.js` to extract DANGEROUS direct-inert
selectors, and fails if any dialog has such a selector in its
ancestor chain UNLESS routed through R244's sibling-iteration
helper.

**Why**: R244 is a specific fix; its tests verify the EXACT call
sites use the new helper. But the NEXT modal added inside an
inerted ancestor would silently break and R244 wouldn't catch it.
R245 closes that generalisation gap with structural reasoning.

**Manual verification**: temporarily injected the OLD buggy JS
pattern (`_safelySetInert(document.querySelector(".container"),
true)`) and confirmed R245 detected `.container` as dangerous and
flagged both dialogs as in its ancestor chain. So R245 would have
caught R240 the day it was committed.

**Audit notes**:
- New "Pattern A++" taxonomy category: cross-file structural
  reasoning beyond pure grep but short of full runtime. Future
  cascade invariants should follow this template.
- Allowlist consistency (R245 в†” R244 helper name) explicitly
  cross-checked via `TestSafeHelperIsKnownToTest`. If someone
  renames `_setContainerSiblingsInert`, both R244 and R245 tests
  fail together в†’ no silent decoupling.
- Limitations explicitly documented: static-only, no runtime DOM
  mutation modeling. Dynamic-modal coverage still depends on
  F-cycle16-playwright (canonical future work).
- Selector matcher is minimal (`.class`, `#id`, plain tag) вЂ”
  sufficient for current codebase. Code comment notes the
  limitation.

**Health**: Excellent. Pioneers a doctrine. Should be referenced
in any future Pattern A++ invariant.
