## 2. 维度突破

### 2.1 R348 — v3.6 perf-baseline 首个 HTTP endpoint 应用

v3.6 perf-baseline 系列前 7 应用都聚焦**单次操作 timing** (heartbeat /
retry / queue / TTL / lock timeout / cold start)。R348 首次锁定 **HTTP
endpoint 响应时间**:

- empty POST < 100ms median (5 runs)
- 1KB POST < 200ms median (5 runs)

之前没有 endpoint 级 perf baseline 意味着 Flask app 内任何 sync I/O leak
或 markdown render 退化都不会被 CI 立刻捕获。R348 关闭了这个口子。

### 2.2 R350 — i18n 语义维度首次锁定

项目已有 45 个 i18n 测试, 全部聚焦**结构** (key set / type / placeholder /
ICU)。R350 首次锁定**语义/内容质量** — untranslated keys 比例 ≤ 5% (实
际 1.57%, 5 个合理保留全在白名单)。

R350 与 doc-parity 系列 (R335/R340/R346) 同源 — 都是 "形状一致 ≠ 内容
正确" 的反思。**新建 i18n consistency 维度**, 未来可扩展:

- ICU plural rule cardinality parity
- Translation length variance audit
- HTML/markup parity in localized strings

### 2.3 R352 — v3.8 test-isolation 6th 跨域应用 + 工业化 future-guard

R352 通过 AST 静态分析**主动发现** 5 个潜在 test pollution 风险:

| Cache                  | 文件                  | 风险类型               |
| ---------------------- | --------------------- | ---------------------- |
| `_BUILD_INFO_CACHE`    | server.py             | 进程级单次缓存         |
| `_FEEDBACK_COUNTERS`   | server_feedback.py    | 累计 counter           |
| `_sse_stats_cache`     | server.py             | TTL cache              |
| `_recent_logs_cache`   | server.py             | TTL cache + key cache  |
| `_config_cache`        | service_manager.py    | TTL cache + generation |

每个都新增了 `reset_*_for_testing()` helper。同时 AST 静态分析**全
codebase** 扫描所有 `_NAME_CACHE` / `_NAME_COUNTERS` pattern, 强制每
个都有 reset 或在显式豁免名单 (4 个真正 frozen 的 lookup dict 豁免,
`__all__` 智能 filter)。

这是 cycle 内**真实预防性 bug fix** (虽然没有当前已知 flake, 但消除了
未来 mock subprocess / counter assertion 的 flake 滋生土壤)。
