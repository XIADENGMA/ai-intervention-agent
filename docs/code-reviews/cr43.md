### §2.3 Production ship review: Track A PWA install prompt (commit 2)

**Scope**: JS module (~205 LoC) + 2 HTML buttons +
~95 LoC CSS + i18n (4 keys × 3 locales) + Flask asset
version wiring + 30 regression tests + pseudo locale +
minified CSS + brotli/gzip precompressed artifacts.

**Quality signals**:

| Aspect | Evidence | Verdict |
|---|---|---|
| Architecture isolation | Pure IIFE module; no `multi_task.js`/`app.js`/`state.js` deps | ✅ zero-intrusion |
| Storage schema | `aiia.pwaInstallDismissed.v1` envelope; consistent with R130/R137/R139 naming | ✅ versioned + future migration path |
| Graceful failure | All `localStorage` ops wrapped in `try/catch silent` | ✅ Safari-private-mode / quota / cookie-disabled safe |
| Event lifecycle | `beforeinstallprompt` capture + `appinstalled` cleanup + `userChoice` accepted/declined branching | ✅ all 3 states handled |
| Dismissal UX | 30-day TTL + auto-clear on `appinstalled` (re-install path) | ✅ no permanent blocking |
| Non-PWA-eligible fallback | Default `hidden`+`disabled`; `beforeinstallprompt` doesn't fire on iOS Safari/Firefox/installed | ✅ button only appears when actionable |
| i18n | 4 keys × 3 locales (en, zh-CN, zh-TW); pseudo locale regenerated; orphan-key test green | ✅ standard compliance |
| Build artifacts | `main.min.css` + `.gz`/`.br` precompressed all synced via `precompress_static.py` | ✅ all build-freshness gates green |
| Test coverage | 30 tests across 6 layers (HTML / JS / CSS / i18n / asset version wiring / PWA prereqs) | ✅ comprehensive |
| ty type-check | Initial failure on `re.search()` narrowing; fixed by switching `assertIsNotNone(m)` → `assert m is not None` | ✅ ty narrowing recognized |

**Design decision review — "purple vs neutral for main button"**:
explicit visual hierarchy choice. Settings (neutral white)
is the **routine** affordance; install (brand purple
`#8b5cf6`) is a **discoverable opportunistic CTA**. The
dismiss cross (28x28 grey) deliberately undersizes vs the
40x40 main button so "later" doesn't compete with the
primary action. **Defensible** — matches design system
hierarchy principles.

**Design decision review — "30-day dismissal TTL"**:
explicit anti-nag UX. Permanent dismiss would prevent
re-prompting when user changes mind; daily prompt would
annoy. 30 days is a reasonable middle ground (PWA install
opportunity surfaces ~12 times/year if user remains
non-installer). Defensible; no concrete data to argue
otherwise.

**Concerns / follow-ups** (none blocking):

- **[low]** No e2e test of actual `beforeinstallprompt`
  → click → install flow. Would require Puppeteer/
  Playwright headed Chrome. Defer per cr39 §8 #7.
- **[low]** iOS Safari standalone install (`Add to Home
  Screen`) has no `beforeinstallprompt` event — user
  must use Share menu. Could show iOS-specific banner;
  defer pending iOS user demand signal.
- **[info]** Storage schema version is `v1` but the JSON
  envelope uses `{dismissed_at, schema_version: 1}`
  redundantly with the localStorage key suffix. Minor;
  could simplify to either-or in future schema rev.

## §4 Lessons learned (cycle-level, not §5.1)

1. **Process gates take effect when codified into copy-
   paste-friendly templates**. cr42 proposed v3.2 in
   prose; cr43 §0.1 boilerplate is what makes it actually
   adopted by future cycle leads. Templates > prose for
   process enforcement.
2. **Track-classification table in §3.3 should be
   standard**. Distinguishing "still deferred"
   (gated) from "bandwidth-deferred" (bonus) clarifies
   what future cycles need to revisit vs what's fully
   parked.
3. **Single-cycle execution × 2 in a row** suggests this
   may be the new normal when carry-over stacks are
   thick. Don't artificially split work into multiple
   cycles for ceremony.
4. **`rg` pre-check can surface architectural context,
   not just discoverability hits**. Track B's verdict
   wouldn't have been possible without the `rg` evidence
   at `server.py:569`. v3.2 is more useful than
   originally framed.
5. **ty static type-checker requires `assert m is not
   None` for `re.Match` narrowing**, not `unittest.assertIsNotNone`.
   Future regex-based tests in this codebase should
   default to `assert` for narrowing to keep ty green.
