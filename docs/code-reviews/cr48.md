## §2 Per-commit review

### §2.1 `c20fd86` Cycle-2 Track A (text-tertiary) — quality **excellent**

**Scope**: `--text-tertiary` token bumped from
1.78:1/2.11:1 (FAIL both) to 3.74:1/4.44:1 (AA-large)
in light, 3.15:1/3.59:1 (AA-large only) to 5.30:1/6.04:1
(AA-normal) in dark. 10 invariants.

**Quality**:
- ✅ **Computation-first tests** — directly parse CSS
  hex values + recompute WCAG ratios. Allows future
  designer tuning within threshold; survives WCAG 3.0
  APCA migration with 1 helper change.
- ✅ **Anti-misuse invariants** — `--text-tertiary`
  allowlisted to strikethrough only (1 hit verified);
  `--text-muted` outright banned from `color:` usage
  (background-only token).
- ✅ **Brand constraint documented** — light bg-primary
  `#e8e6dc` (Anthropic warm beige) cannot reach AA-
  normal for muted text; AA-large + strikethrough is
  WCAG-permissible compromise. Decision rationale in
  CSS comment + cycle-2 §3.

**Concerns**: none.

### §2.2 `c31b475` Cycle-2 Track B + latent-bug fix — quality **outstanding**

**Scope**: 4 status colors × 2 bgs × 2 themes = 16
cells audited. 6 status color values updated (success
+ warning + error + info in light, error + info in
dark) for AA-normal compliance. **Latent bug discovered
+ fixed**: `@media (prefers-color-scheme: light)` block
didn't mirror status colors → system-light users (no
explicit `data-theme="light"`) were getting dark-theme
status colors on light bg = 8/8 FAIL.

**Quality**:
- ✅ **Discovery > declaration** — Track B started as
  "audit status colors" but invariants for the media-
  query mirror surfaced a pre-existing functional bug.
- ✅ **5 mirror-presence invariants** — for each status
  token in the media-query block. Prevents future
  divergence.
- ✅ **Cycle-2 §6 L5 generalization** — "audit
  cascading overrides" formalized as a methodology
  step. Will inform cycle-3+ and future audits.
- ✅ Borderline-FAIL revealed mid-test — initial color
  picks (`#5e7048` / `#965a07` / `#3a6f9f`) were
  4.32/4.46/4.24 on bg-primary which is just under
  AA-normal 4.5; tests caught + drove a 2nd iteration
  to `#506840`/`#825005`/`#2e5e8c` (4.94/5.42/5.42
  comfortable margin).
- ✅ Status color updates preserve hue family — only
  lightness decreased; brand identity intact.

**Concerns**:
- 📌 Status colors used 5x in CSS with various `.error
  / .success` text + bg combinations — only `:root` /
  `data-theme="light"` token defs were checked. If any
  component uses `color: #ef4444` hardcoded (instead
  of `var(--error-500)`), it bypasses this fix. cycle-4
  could add an invariant "no status hex hardcoded
  outside :root scopes". **Defer**: not regression-
  worthy this cycle.

### §2.3 `0b5e82f` Cycle-3 Track A (focus ring) — quality **excellent**

**Scope**: WCAG 1.4.11 (Non-text Contrast) audit
found light `--primary-500 #d97757` focus ring at
2.50:1/2.96:1 (FAIL). New single-purpose
`--focus-ring-color` token introduced. `:focus-visible`
rule uses triple-fallback `var(--focus-ring-color,
var(--primary-500, currentColor))`.

**Quality**:
- ✅ **Semantic separation** — focus is distinct from
  brand (cycle-3 §6 L1 "brand color ≠ focus color").
  Prevents future re-conflation.
- ✅ **Margin-aware color pick** — `#b35a3c` chosen at
  3.78/4.49 instead of `#c56a4c` (primary-600) at
  3.03/3.60 borderline. Renderer gamma variations
  won't tip it under threshold.
- ✅ **Triple-fallback chain** — backwards-compat for
  custom themes, no breaking change for callers.
- ✅ **Test extraction fix mid-iteration** — initial
  regex naively grabbed first `:focus-visible` in CSS
  (which is `.upload-btn-label:focus-visible` at line
  2648); fix narrowed scope to `@layer a11y` block
  specifically using brace-counting + lookbehind for
  `(`. **Demonstrates test robustness culture**.

**Concerns**:
- 📌 8 component-specific `:focus-visible` rules
  (lines 2648, 3347, 3685, 3756, 3819, 3852, 3965)
  each have their own outline colors — they're not
  using `--focus-ring-color`. If any of them use light
  primary-500, same a11y issue exists. **cycle-4
  candidate**: per-component focus ring contrast
  sweep.

### New methodology patterns established this cr cycle

   — recompute WCAG ratios from CSS hex values rather
   than snapshot. Survives threshold formula evolution.
2. **Audit cascading overrides** (cycle-2 §6 L5)

   (`:root`, explicit `[data-theme]`, media queries),
   not just the primary scope.

   — semantic-purpose tokens prevent a11y traps.
