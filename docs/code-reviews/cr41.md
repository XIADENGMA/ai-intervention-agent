### Methodology v3.1 column proposal

For cycle-6 mining doc kickoff, add column to §0 mandatory
columns table:

```
4. **NEW: borrow kind** — one of:
   - **schema** — direct MCP tool / route / data-shape borrow
   - **inspiration** — UX pattern only, no schema mapping
   - **N/A** — not borrowable
```

Rationale: schema borrows demand subject-type-1-to-1
matching (MCP server to MCP server). Inspiration borrows
can cross subject-type boundaries freely. Explicitly
labeling each candidate prevents Continue.dev-style
"survey IDE plugin for schema" mis-categorization.
