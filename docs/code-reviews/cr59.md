## 4. 风险 & 隐患

### 4A. v3.6 pattern 覆盖密度差异

R296 / R297 / R298 / R299 / R300 5 个 pattern 都是「首次落地」, 当前
仅 1-2 个 case 验证。后续 cycle 若想推广 (e.g. perf-baseline 扩展到
Bark retry / image-upload chunk size), 需要再迭代 2-3 个 case 才算
**pattern 稳定**。

**风险等级**: 低
**缓解建议**: cycle-30 选 1-2 个 v3.6 pattern 做 second case (e.g.
perf-baseline 第二轮 / lifecycle-cleanup 扩展到 dialog modal 的 escape
key listener), 把 pattern 从 "首次" 推到 "稳定"。

### 4B. Architecture diagram 维护负担

R300 引入了 12 节点 + 12 箭头的 Mermaid component diagram, **未来加新
组件时需要手动更新双语 README + R300 invariant**。R300 invariant 锁了
"必须列 12 节点", 没锁 "未来新组件必须入图"。

**风险等级**: 中
**缓解建议**: cycle-30/31 加 meta-invariant 测试 — 扫描
`src/ai_intervention_agent/web_ui_routes/` 目录的模块数, 与 Mermaid
diagram 的 Backend 节点数对比, 若 Backend module 新增超过 +2 应当
触发 "请更新 architecture overview" 提醒。但这个 meta-invariant 复杂度
高 (要识别 "module 边界"), 可能不值得做。**优先方案**: review checklist
里加一行 "添加新 web_ui_route / notification subclass 时记得同步
README architecture overview"。

### 4C. cross-language schema (R297) 当前只 cover SSE

R297 只 cover SSE event payload (Python `EVENT_SCHEMAS` ↔ JS
`addEventListener`), **REST API response schema 仍未 cross-language
校验** (e.g. `GET /api/tasks` Python 返回 dict ↔ JS `fetch().then(data
=> data.X)` 字段访问)。这是 v3.6 cross-language pattern 的下一个
应用目标。

**风险等级**: 中
**缓解建议**: cycle-30 R301 候选 — REST API response schema cross-language
invariant (用 Python `TypedDict` / Pydantic schema 描述 response, 反向
校验 JS handler 字段访问)。
