### §3.2 `d74587f` — mining-3 borrow #1 header chip (full-stack)

**Quality**: ⭐⭐⭐⭐⭐ best-in-class full-stack feature commit.

- **Schema**: `Task.header_label: str | None = None`, MCP
  `Field(...)` with explicit 16-char description, examples,
  cycle reference, AT semantics
- **Server normalization**: `add_task` `strip + clamp +
  empty→None` via `HEADER_LABEL_MAX_LENGTH = 16` constant
- **Persistence**: `_persist` + `_restore` round-trip preserved
- **Frontend**: `updateHeaderChip(label)` helper + double call
  site (`switchTask` cache path + `loadTaskDetails` async path)
- **HTML**: `#task-header-chip` anchor with `role="status"` +
  `aria-label` (conditional, only when label present)
- **CSS**: pill style, primary-color outline, uppercase letter-
  spacing — visually distinguishable but unobtrusive
- **Tests**: 21 cases / 8 classes covering model / clamp /
  persistence / MCP schema / route / frontend / HTML / CSS
- **Mock compat**: `test_web_ui_routes.py` mock tasks updated

The **double slice 0..16 (server + client)** is a defensive
pattern — server normalizes, but client also clamps in case of
restored snapshot or out-of-date server. Same as cr37 §8 #1's
constant strategy: belt + suspenders.

UUID-based task IDs in `TestRoute` setUpClass avoid cross-test
DB pollution. Lessons learned from earlier `test_post_accepts_*`
collisions in `test_feat_mining3_yesno_type.py`.

### §4.3 Hot-path performance impact

- `Task.header_label` is `Optional[str]`, default `None` —
  zero overhead when feature unused (~99% of tasks).
- `updateHeaderChip` does 1 DOM query (`getElementById`) per
  task switch — same cost as `updateYesnoButtonGroup`. **Total
  switchTask cost grew from N=2 DOM operations to N=3** for the
  three borrowed features combined. Below noise threshold.
- CSS class `.task-header-chip` is `display: none` when hidden;
  no layout pass cost.

### §4.4 Accessibility audit

- `task-header-chip` exposes `role="status"` + `aria-label` —
  AT users hear chip on task switch
- yesno hide-textarea now `aria-hidden=true` + `tabindex=-1` —
  Safari + VoiceOver < 16 bug guarded
- 44px touch target on yesno buttons (WCAG 2.5.5) — preserved
- 16-char clamp on header label ensures chip doesn't wrap or
  truncate visually

### §4.5 Backward compatibility

- All 3 new MCP fields (`feedback_placeholder`, `question_type`,
  `header_label`) are optional, default `None`.
- Persistence `_restore` handles missing fields gracefully
  (`item.get(...)` not `item[...]`).
- API responses include new fields; old clients ignore unknown
  keys per JSON contract.
- **Safe for v1.8.0 minor bump**.
