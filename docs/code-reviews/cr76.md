**R406 新维度的真实失败防御**:
1. **Orphan setup method** — 新增 Mixin 但忘记在 `__all__` 注册;
2. **Orphan __all__ entry** — `__all__` 列出 Mixin 但 `WebFeedbackUI` 父类
   列表里漏继承, 或 setup 方法漏调用;
3. **Drift between __all__ / parent classes / setup calls 三方** — 三个清
   单逐渐 drift, 导致某些 route 注册 / 某些不注册, 出现 404 静默失败;

**invariant 价值层级**: R406 防御的不是单个 route 工作正常 (这是 unit
test 的范畴), 而是 "应该注册的 route 全部都注册了" (覆盖率层级)。这是
invariant 区别于 unit test 的核心价值。

**与 R398/R404 形成 OpenAPI 三层保护矩阵**:
- **架构层** (R406): route 必须存在 → 通过 Mixin registration 矩阵保证;
- **schema 层** (R398): 存在的 route 必须有完整 schema → properties.type;
- **可读性层** (R404): schema 必须有高质量 summary → endpoint first-line;
